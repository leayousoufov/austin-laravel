Facing Your Demons

  • Upgrade from 4.2 to 5.3
  • Implementing API authentication
  • Implementing social logins
Again, Laravel isn't backwards compatible so the upgrade was a disaster. Almost everything broke
Authentication without Laravel & everything it provides would have been a nightmare - a secondary package would have been required and what are the chances that something like that gets maintained properly?
Social logins would also require a package and would get unbelievably messy and untameable.

Tending to Your Demons

  • Laravel's extensive documentation
  • The community
  • Laravel Passport & Auth facade
  • Laravel Socialite
The extensive documentation carefully traced all the steps necessary for the upgrade
Whatever the documentation lacked - the community helped with - your question has already been answered
For authentication - Laravel offers two amazing packages (Passport & Socialite)
Passport - provides a full OAuth2 server implementation for API authentication (announced in 2016 at Laracon)
Socialite - fluent interface to OAuth authentication with FB, G+, Twitter, more
Whenever the FB API changes - Socialite changes with you, we no longer have to worry about our social logins breaking with every update.
